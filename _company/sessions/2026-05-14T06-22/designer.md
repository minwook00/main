🎨 Designer: 작업 시작합니다.

요청하신 프롬프트 2에 따라, '긴장감과 몰입감'을 극대화하고 심리적 장치(자기효능감 자극, 희소성)를 시각적으로 구현하는 React/Tailwind CSS 기반의 UI/UX 디자인 시안을 구성합니다.

이 디자인은 사용자가 서비스에 접속했을 때 **'당장 참여해야 한다'**는 긴박감과 **'나 자신에 대해 돌아보게 하는'** 내적 동기 부여를 동시에 제공하는 데 중점을 둡니다.

## 🖼️ 디자인 시안: 취미 기반 매칭 플랫폼 (Frontend Mockup)

### 1. 디자인 시스템 정의 (Brand Identity & Palette)

브랜드 정체성(전문적이면서 친근함)과 긴장감을 표현하기 위해 어두운 모드 기반의 다크 테마와 대비되는 포인트 컬러를 사용합니다.

| 요소 | 값/설명 | 목적 |
| :--- | :--- | :--- |
| **Primary Color (Dark)** | `#1A1A2E` (Deep Indigo) | 배경, 깊이감 부여. 몰입감을 높임. |
| **Secondary Color (Accent)** | `#FF6B6B` (Coral Red) | 긴급성, 흥미 유발(CTA), 호감 표시의 강조. |
| **Tertiary Color** | `#4D96FF` (Sky Blue) | 정보 전달, 차분함, 신뢰성 부여. |
| **Text Color** | `#E0E0F8` (Off-White) | 가독성 확보. |
| **Typography** | **Inter** 또는 **Pretendard** | 전문적이면서도 친근한 산세리프체. |
| **Shadow/Depth** | Subtle, diffused shadows for depth and separation. | 요소 간의 분리와 입체감을 부여하여 고급스러운 느낌 강조. |

### 2. 메인 화면 시안 (Website Main Screen)

**목표:** 사용자의 현재 관심사와 시간적 희소성을 즉각적으로 제시하여 클릭을 유도합니다.

#### 🌟 컴포넌트 구조: `MainDashboard`

| 영역 | 구성 요소 | 디자인 특징 및 UX 전략 |
| :--- | :--- | :--- |
| **Header** | 로고, 내비게이션 (홈, 마이페이지) | 심플하게 유지. 상단 우측에 알림/프로필 아이콘 배치. |
| **Focus Area** | **Active Theme Cards Grid** | 카드 형태를 사용하여 시각적 집중도를 높임. 3~4개의 주요 주제(게임, 영화, 직업 등) 카드를 격자 형태로 배치. |
| **Theme Card (핵심)** | **[주제 이름]** | 크고 명확하게 표시. |
| | **⏳ Countdown Timer** | **가장 강조.** `Coral Red`로 강조하며, 남은 시간을 숫자로 크게 표시. (예: `02:15:30`) |
| | **CTA Button** | "지금 참여하기" 버튼. 배경색은 `Accent Color`. 마우스 오버 시 색상 변화(Hover effect)를 통해 즉각적인 반응 유도. |
| **Psychological Hook** | (선택 사항) | 카드 하단에 작은 텍스트로 "당신의 취미 자부심을 확인하세요." 등의 문구를 추가하여 자기효능감 질문을 간접적으로 삽입. |

**레이아웃 예시 (Tailwind CSS 기반):**
*   배경: `bg-[#1A1A2E]` (Deep Indigo)
*   카드 배경: `bg-[#282A3D]` (Slightly Lighter Dark Gray)
*   타이머 텍스트: `text-4xl font-bold text-[#FF6B6B]`

### 3. 채팅 대기실 랜딩 페이지 시안 (Waiting Room Landing Page)

**목표:** 사용자가 링크를 타고 들어왔을 때 '제대로 된 곳에 왔다'는 느낌과 함께 **자기효능감 자극 질문**을 통해 몰입을 시작합니다.

#### 🌟 컴포넌트 구조: `WaitingRoomLanding`

| 영역 | 구성 요소 | 디자인 특징 및 UX 전략 |
| :--- | :--- | :--- |
| **Title Area** | 환영 메시지, 현재 테마 표시 | 차분한 파란색 계열(`Sky Blue`)로 신뢰감을 부여. |
| **Self-Efficacy Question (핵심)** | **"당신의 취미 자부심은 몇 점인가요?"** | 중앙에 매우 크게 배치. 질문 자체를 카드 형태로 감싸 몰입도를 높임. |
| **Interaction Area** | **Interactive Slider/Input** | 사용자가 자신의 가치관이나 수준을 스스로 평가하도록 유도하는 슬라이더(1점~10점) 또는 텍스트 입력 필드. 이 과정이 곧 '준비 과정'임을 인지시킴. |
| **Countdown Indicator** | **[남은 시간 표시]** | 상단에 고정하여 긴장감을 유지. "대화 시작까지 남은 시간: [T-X시간]"을 명확히 표시. |
| **Entry Point** | **[참여하기 버튼]** | 이 버튼은 단순히 채팅방으로 이동하는 것이 아니라, '나의 자부심'이라는 행동에 대한 보상임을 강조해야 함. (Accent Color 사용) |

**UX 흐름:**
1.  사용자 접속 $\rightarrow$ 화면 중앙에 큰 질문 제시 $\rightarrow$ 스스로 답을 고민하거나 입력 $\rightarrow$ **참여하기** 버튼 클릭 $\rightarrow$ 다음 단계(실제 채팅방 진입).

### 4. 호감 표현 UI (Mutual Like Screen) 디자인 시안

**목표:** '거절당할 공포'를 최소화하고, 상호 간의 선택이 공개되는 순간에 감정적 몰입을 극대화합니다.

#### 🌟 컴포넌트 구조: `VoteScreen`

| 영역 | 구성 요소 | 디자인 특징 및 UX 전략 |
| :--- | :--- | :--- |
| **Psychological Safety Banner (필수)** | **"상대방은 당신이 선택했는지 알 수 없습니다. 서로 선택했을 때만 공개됩니다."** | 화면 상단에 붉은색 또는 강조된 경고 박스로 고정 배치하여, 시스템의 심리적 장치를 명시적으로 전달. |
| **Mutual Like Grid (핵심)** | **상대방 목록 그리드** | 카드 형태의 그리드 레이아웃 사용. 사진 대신 **캐릭터 아이콘**을 사용하여 익명성을 유지하고, '사람'이 아닌 '관심사/매력'에 집중하게 함. |
| | **Character Icon** | 각 항목에는 심플한 이모티콘이나 추상적인 캐릭터 아바타를 배치하여 친근함과 비대면의 느낌을 동시에 제공. |
| | **Mutual Like Indicator** | 선택된 상대에게는 `Accent Color`로 하트 아이콘 또는 체크 표시를 명확히 표시. |
| **Action Button** | **[호감 표시하기] / [모두 공개하기]** | '모두 공개하기' 버튼은 높은 긴장감을 유발하며, 사용자가 의식적으로 선택하도록 유도함. |

**디자인 강조점:**
*   **비주얼:** 사진 대신 심플한 캐릭터 아이콘을 사용하여 익명성을 존중하고, 감정적 연결이 외모가 아닌 **'선택의 행위'**에 있음을 강조합니다.
*   **색상 활용:** 선택된 관계는 `Accent Color`로 하이라이트하여 긍정적인 피드백을 시각적으로 즉시 제공합니다.
